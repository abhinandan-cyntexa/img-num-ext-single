# 260421-001 Summary: Single-Card Architecture

## Result

Implemented and verified a static Tableau Viz Extension that renders exactly one image-plus-number KPI card from worksheet data.

## Changes

- Added Tableau runtime files: `index.html`, `styles.css`, and `chart.js`
- Added a minimal Excel test dataset: `test-data/image-number-single-card.xls`
- Added local Tableau manifest: `img-num-ext-single-local.trex`
- Added Tableau Online HTTPS manifest: `img-num-ext-single-https-local.trex`
- Added HTTPS diagnostic manifest and page: `img-num-ext-single-debug-https-local.trex`, `debug.html`
- Added GitHub Pages manifest: `img-num-ext-single-github-pages.trex`
- Added Python HTTPS server: `serve_https.py`
- Added project documentation: `README.md`
- Added GSD planning state under `.planning/`

## Architecture

- The manifest exposes two encodings: `Image URL` and `Value`
- `chart.js` initializes the Tableau Extensions API, reads visual encodings, loads worksheet summary data, and renders only the first row
- Numbers use comma formatting through `Intl.NumberFormat`
- Missing or broken images fall back to the neutral people icon used by the reference shape
- `SummaryDataChanged` triggers a re-render
- A top-left status panel shows current extension state and `Image URL` / `Value` mapping diagnostics
- Local development stays on `python3 -m http.server 8081`; no `package.json` was added
- Tableau Extensions API SDK is vendored locally at `vendor/tableau.extensions.1.latest.js`
- Runtime parser supports current Tableau visual-spec and paged summary-data APIs
- Manifest version is `0.3.0`; runtime structure now matches the working heatmap reference more closely
- Tableau Online uses `https://localhost:8443/index.html` to avoid browser mixed-content blocking
- The HTTPS diagnostic page verifies local source loading and `tableau.extensions.initializeAsync()` before testing `chart.js`
- GitHub Pages uses `https://abhinandan-cyntexa.github.io/img-num-ext-single/index.html` for hosted HTTPS testing
- CSS includes a global `[hidden]` rule so failed image elements do not show the browser's broken-image glyph

## Notes

- Port `8080` was already occupied by another Python process, so this project uses `8081` and the manifest points to `http://localhost:8081/index.html`.
- Tableau Online requires HTTPS; the HTTP manifest is only for Tableau Desktop/local testing.
- GitHub Pages avoids localhost certificate handling once the repo has published.
- Local runtime artifacts `certs/` and `__pycache__/` are ignored and should not be committed.
