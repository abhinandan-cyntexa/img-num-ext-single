# 260421-001 Plan: Single-Card Architecture

## Goal

Build a static Tableau Viz Extension that renders one image-plus-number card from worksheet data, using the referenced multi-card project as the product direction while avoiding grid complexity.

## Outputs

- `index.html`
- `styles.css`
- `chart.js`
- `test-data/image-number-single-card.xls`
- `img-num-ext-single-local.trex`
- `img-num-ext-single-https-local.trex`
- `img-num-ext-single-debug-https-local.trex`
- `debug.html`
- `serve_https.py`
- `README.md`
- `.planning/STATE.md`

## Tasks

1. Create the static extension shell and Tableau manifest with `Image URL` and `Value` encodings
2. Implement data loading, visual-spec parsing, first-row selection, number formatting, image fallback, and re-rendering
3. Add a minimal Excel test dataset and local development documentation
4. Add HTTPS Tableau Online testing support and a diagnostic manifest
5. Verify static server responses, manifest XML, SDK loading, status UI, and fallback rendering

## Acceptance Criteria

- `python3 -m http.server 8081` can serve all runtime files
- `python3 serve_https.py` can serve all runtime files for Tableau Online
- `test-data/image-number-single-card.xls` provides Tableau-ready fields for `Image URL` and `Value`
- `chart.js` does not render a grid or loop cards into the DOM
- Missing/broken image sources show a neutral placeholder
- Fallback state does not show the browser's broken-image square
- `.planning/STATE.md` records the quick task completion

## Notes

- Tableau Online verification was completed manually by the user after the HTTPS manifest/server path was added.
